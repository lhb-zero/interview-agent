# 面试智能助手 (Interview Agent) — 需求分析文档

> 版本：v1.1  
> 日期：2026-04-23  
> 状态：已确认

---

## 一、项目背景

面向求职者的面试智能助手，帮助用户系统化准备技术面试。用户输入岗位方向或技术关键词，AI 自动总结并生成对应的面试题和知识点，支持多领域覆盖（Java、Python、AI 等）。

本项目同时作为 Java 工程师向 AI 应用开发转型的实战练习项目，重点实践 RAG（检索增强生成）、向量化、Tool Calling 等 AI 工程核心技术。

---

## 二、目标用户

- 技术求职者：需要快速梳理某领域面试重点
- 在职工程师：想系统性复习技术知识
- 面试准备者：希望进行模拟面试练习

## 四、非功能需求

| 需求编号 | 需求描述 | 指标 |
|----------|----------|------|
| NF-001 | 响应时间 | 首次 Token 延迟 < 3s（本地 Ollama） |
| NF-002 | 并发能力 | 支持单用户使用（本地部署） |
| NF-003 | 可扩展性 | 模型可切换、Tool 可插拔 |
| NF-004 | 成本控制 | 核心功能全免费（本地部署） |

---

## 五、约束条件

| 约束项 | 说明 |
|--------|------|
| LLM 运行时 | 本地 Ollama，模型 qwen3.5:4b（后续升级 9b） |
| Embedding 模型 | 本地 Ollama bge-m3（中文优化） |
| 运行环境 | Windows 开发机，需安装 Docker Desktop |
| 硬件要求 | 8GB+ 内存，推荐 16GB |
| 成本 | 核心功能免费，多模型切换时部分 API 付费 |

## 六、术语表

| 术语 | 说明 |
|------|------|
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| Embedding | 将文本转换为高维向量表示 |
| Chunking | 将长文档切分为小段文本 |
| Tool Calling | LLM 自主决策调用外部工具的能力（Spring AI 1.0 新特性，替代 Function Calling） |
| Prompt Template | 外部化管理的提示词模板（.st 文件） |
| PGVector | PostgreSQL 的向量搜索扩展 |
| BGE-M3 | 智源研究院的多语言多粒度 Embedding 模型 |
| SSE | Server-Sent Events，服务端推送流式响应 |
