| 服务 | 端口 | 说明 |

| Spring Boot 后端 | 2180 | application.yml 配置 |

### 2.3 Docker 配置

- **PostgreSQL + PGVector**: `docker-compose.yml`
- 容器内端口 5432 → 宿主机端口 **5600**
- 数据库名: `interview_agent`，用户: `interview_agent`，密码: `interview_agent_2026`
- 初始化 SQL: `interview-agent-dao/src/main/resources/schema.sql` 挂载为 `/docker-entrypoint-initdb.d/01-schema.sql`
- Docker 镜像源配置: `C:\Users\Zero\.docker\daemon.json`

- **TEI Reranker**: `docker-compose.yml`
- 容器内端口 80 → 宿主机端口 **18081**（8081 被 Hyper-V 保留）
- 模型: `bce-reranker-base_v1`（279MB，bge-reranker OOM 后换用）
- 首次启动自动下载模型（约 1.1GB），后续使用 Docker volume 缓存
- 健康检查: `GET /health`，启动等待 120s（模型加载时间）
- CPU 模式运行，不占用 GPU VRAM

## 五、开发计划与路线图

### 5.1 RAG 优化路线图

| 阶段 | 优化项 | 效果 | 状态 |
|------|--------|------|------|
| Phase 1 | Re-ranking（TEI + bce-reranker-base_v1 Cross-Encoder） | 检索精度提升 | ✅ |
| Phase 2 | 混合检索（PGVector + ILIKE + RRF 加权融合） | 召回率大幅提升 | ✅ |
| Phase 3 | 查询改写（QueryRewriteService + Prompt Rewrite） | 模糊提问改善 | ✅ |
| Phase 4 | 迁移到 RetrievalAugmentationAdvisor 架构 | 架构模块化 | 待开发 |
| Phase 5 | 上下文压缩 + RAG 评估 | Token 效率 + 量化 | ✅（评估部分） |

> 详细设计见 `docs/RAG实现与优化指南.md` 和 `docs/RAG/RAG评估模块技术文档.md`

### 5.2 DeepSeek API 集成架构

```
Service → ChatModel → DeepSeekChatModel (wrapper) → OpenAiChatModel → Interceptor 注入 thinking → DeepSeek API
```

- 核心：`chat/deepseek/` 包（4 文件）+ `ChatOptionsFactory` + `ChatModelConfig`
- 详见 `docs/技术指导/DeepSeek API 集成技术选型与实现文档.md`
